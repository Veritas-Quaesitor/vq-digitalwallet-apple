Prerequisites

Apple Pay on the web requires all four of the following to process a live payment.

📱

Apple device with Safari

Safari on macOS or iOS, or any iOS 18+ browser via Apple's QR-based payment relay. Apple Pay will not fire on non-Apple hardware.

🪪

Apple Merchant ID

Register a Merchant ID in the Apple Developer portal. Format: merchant.com.yourcompany

🌐

Verified HTTPS domain

Apple verifies domain ownership via a signed file served at /.well-known/apple-developer-merchantid-domain-association over HTTPS. This is a one-time setup step per domain.

🔒

Server-side validation endpoint

A backend HTTPS endpoint on your domain that proxies Apple's mTLS merchant validation handshake using your Apple Pay merchant certificate. See the flow below.

Quick Start

Full integration in under 25 lines — including optional payer data and shipping fields.

JavaScript 
// npm:  const VqDigitalWalletApple = require('vq-digitalwallet-apple');
// CDN:  window.VqDigitalWalletApple (after loading vqdigitalwalletapple.js from unpkg)

const applePay = new VqDigitalWalletApple({
  mode:                  'production',
  merchantIdentifier:   'merchant.com.yourcompany',
  merchantName:         'Your Store',
  validationEndpoint:   'https://your-server.com/applepay/validate',
  allowedCardNetworks:  ['visa', 'masterCard'],
  merchantCapabilities: ['supports3DS'],
  buttonStyle:          'black',
  buttonType:           'buy',
  paymentOptions: {
    requestPayerName:      false,
    requestPayerEmail:     false,
    requestPayerPhone:     false,
    requestBillingAddress: false,
    requestShipping:       false,
    shippingType:          'shipping'
  },
  onTokenGenerated(token, error) {
    if (error) { console.error('Payment failed:', error); return; }
    // Send token to your backend for authorisation
    fetch('/api/process-payment', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ token })
    });
  }
});

const isReady = await applePay.initialize();
if (isReady) {
  applePay.createButton('pay-button-container', {
    amount:      99.99,
    currency:    'ZAR',
    countryCode: 'ZA'
  });
}

Merchant Validation Flow

Apple Pay requires a server-side mTLS handshake before the payment sheet is shown. Your backend acts as the relay.

1
Browser

User clicks the Apple Pay button

The SDK calls PaymentRequest.show(). The browser fires a merchantvalidation event containing a one-time Apple-issued validationURL.

2
SDK

SDK POSTs to your validationEndpoint

The SDK extracts the validationURL, validates it against *.apple.com, then POSTs { validationURL, merchantIdentifier, displayName, domainName } to the endpoint you configured.

3
Your server

mTLS request to Apple

Using your Apple Pay merchant certificate, your server makes a mutual TLS request to Apple's validationURL and receives a signed, time-limited merchant session object.

4
Your server

Merchant session returned to SDK

Your endpoint returns the raw merchant session JSON. The SDK calls event.complete(session) to unlock the Apple Pay sheet for the user.

5
Browser

User authorizes payment

The Apple Pay sheet presents the amount, merchant name, card selector, and a biometric authentication prompt (Face ID / Touch ID). The user does not enter card details manually.

6
SDK

onTokenGenerated called with payment token

The SDK Base64-encodes the encrypted payment token and calls your onTokenGenerated callback. Forward the token to your payment processor for authorisation and settlement.

Server-side merchant validation is outside the scope of this SDK. See Apple's Merchant Validation documentation for the mTLS implementation guide.
